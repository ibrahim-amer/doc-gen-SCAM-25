The purpose of this part of the questionnaire is to assess the documentation generation for the Make language.

In general, a makefile contains rules of the form:

target : prerequisites
    recipe

where target is the name of a file, prerequisites are the dependencies of the file (if any), and recipe is the sequence of shell commands.

Many of our examples contain “phony” targets, where the target is not a file name, but instead serves as a name for the recipe. For phony targets, the list of dependencies often is empty (meaning that the target is always out‑of‑date).

Below is a header comment for a rule with a phony target called do-push:

#@@ Pushes Docker image to a repository with version and latest tags if conditions are met.
#@ The "do-push" command pushes the Docker image with the specified version tag to the repository. If the "TAG_WITH_LATEST" variable is not set to "never" and either set to "always" or the "BASE_RELEASE" matches the "VERSION", it also pushes the image with the "latest" tag.
do-push:
    docker push $(IMAGE):$(VERSION)
    @if [[ $(TAG_WITH_LATEST) != never ]] && ([[ $(TAG_WITH_LATEST) == always ]] || [[ $(BASE_RELEASE) == $(VERSION) ]]); then \
        echo docker push $(IMAGE):latest >&2; \
        docker push $(IMAGE):latest; \
    fi

Here, the header comment contains a short description (prefixed #@@) and a longer description (prefixed #@).
